Fullstack MERN Application

This is a Create, Read, Update, Delete app created with the MERN stack, which includes MongoDb, ExpressJS, ReactJS, and NodeJS. I wrote the frontend and backend of this app in the first two weeks of 2024. This was my first time using a NoSQL Database instead of using postgreSQL.

This application allows the user to sign-up or sign-in and add a location to the app. This sign in includes Authorization using JSON Web Token and the password on the DB is encrypted using bcryptJS.

When signed in the user can Create a new 'place', Update or Delete their existing 'places'. The user can also open up a Google Maps view of any place on the app using the Google API.

If the user is not signed in they can still view other users 'places' and see them on the map.

Solving a System Persistence issue using Amazon S3 Cloud Storage

I had a previous bug within the backend. Heroku as a service will shut down the server occasionally. Once servers reset the images stored will be deleted.

Ive fixed this by using Amazon S3 Cloud Storage so all pictures used are uploaded to Amazon S3 instead of using FS from Node to save them to the Heroku Server. Now when the servers are temporarily shut off due to inactivity I can continue using files uploaded by users.

Rest API built with Go

This is an API built using Go and SQLite3. I wanted to build a backend project using something different from Node.js and I really liked using Go previously for the Monkey Interpreter.

Libraries Used

• Gin library which was very helpful for HTTP request routing.

• jwt-go for Auth tokens in requests to verify users.

• bcrypt for encrypting and decrypting passwords stored in the SQL DataBase.

• go-sqlite3 for running the SQL server.


API Requests Supported


GET Requests

api/events

• Get a list of available events

api/events/21

• Get a specific event based on ID


POST Requests

api/events (Auth Required)

• Create bookable event

api/signup

• Create a new user

api/login (Returns Auth Token)

• Sign in and Authenticate user

api/events/21/register (Auth Required)

• Register the user for an Event


PUT Requests

api/events/21 (Auth Required) (Only by Creator)

• Update an existing Event


DELETE Requests

api/events/21 (Auth Required) (Only by Creator)

• Delete an Event by ID

api/events/21/register (Auth Required)

• Delete a registration for an Event
